# BI Publisher MS Word Reports (`.rtf`) Guide

## Part 2: MS Word BI Publisher Report (`.rtf`)
Your requirements state: *"The mentor prefers to see the report in Word with BI Publisher... accept input parameters for employees and department from list of values."*

### Step 2.1: Generate the XML Data Source (Report Builder)
To power a BI Publisher Word document, you first need raw XML data.
1. Open **Oracle Reports Builder**.
2. Connect to the Database.
3. Open the **Data Model** view. Click the **SQL** icon and click on the canvas.
4. Paste the following Query. *Note that it uses two parameters precisely matching your requirement!*

```sql
SELECT e.EMPLOYEE_ID, 
       e.EMPLOYEE_FIRST_NAME, 
       e.EMPLOYEE_LAST_NAME, 
       e.EMPLOYEE_TYPE, 
       d.DEPARTMENT_NAME
  FROM HMS_EMPLOYEES_<SUFFIX> e
  JOIN HMS_DEPARTMENT_<SUFFIX> d ON e.DEPARTMENT_ID = d.DEPARTMENT_ID
 WHERE d.DEPARTMENT_ID = :P_DEPARTMENT_ID
   AND (e.EMPLOYEE_ID = :P_EMPLOYEE_ID OR :P_EMPLOYEE_ID IS NULL)
```
5. Go to **File → Generate to File → XML**.
6. Save this file to your Desktop as `HMS_EMP_DATA.xml`. (This is just sample data to train Microsoft Word).

### Step 2.2: Design the Template in Microsoft Word
1. Open **Microsoft Word**. Ensure you have the **BI Publisher** Add-in tab installed.
2. In the BI Publisher ribbon, click **Sample XML**. Select the `HMS_EMP_DATA.xml` file you just saved.
   *(Word will show "Data loaded successfully").*
3. Click the **Table Wizard** button in the ribbon.
4. Select **Table** → Next.
5. Select the grouping (usually `G_EMPLOYEE_ID` generated by your query) → Next.
6. Select all the fields (First Name, Last Name, Type, Dept) to display in your table → Next → Finish.
7. Microsoft Word will generate a shaded XML table. You can now use standard Word tools to color it, add your Hospital Logo at the top, and title it "Department Employee Report".
8. **Save As:** `HMS_EMP_REPORT_<SUFFIX>.rtf`.

### Step 2.3: Register in Oracle EBS XML Publisher
Now we map your Microsoft Word template to your EBS Concurrent Program!
1. Log into Oracle EBS. Switch Responsibility to **XML Publisher Administrator**.
2. Navigate to **Data Definitions** → Click **Create Data Definition**.
   * **Name:** `HMS Employee Report <SUFFIX>`
   * **Code:** `HMS_EMP_RPT_<SUFFIX>` *(CRITICAL: This MUST exactly match the Short Name of your Concurrent Program!)*
   * **Application:** `Application Object Library`
   * Click **Apply**.
3. Navigate to **Templates** → Click **Create Template**.
   * **Name:** `HMS Employee Template <SUFFIX>`
   * **Code:** `HMS_EMP_RTF_<SUFFIX>`
   * **Data Definition:** Select the Data Definition you just created above!
   * **Type:** `RTF`
   * **File:** Upload your `HMS_EMP_REPORT_<SUFFIX>.rtf` Word document.
   * Click **Apply**.

### Step 2.4: Finalize the Concurrent Program Execution
1. Switch back to **System Administrator** responsibility.
2. Go to **Concurrent → Program → Define** and find your `HMS Emp Dept Report <SUFFIX>` program.
3. Change the **Output Format** from `Text` to `XML`! *(If it remains Text, BI Publisher won't trigger).*
4. Add your two parameters:
   * Seq 10 | Parameter `P_DEPARTMENT_ID` | Value Set: `HMS_DEPT_LOV_<SUFFIX>`
   * Seq 20 | Parameter `P_EMPLOYEE_ID` | Value Set: (Create a new fast LOV for Employees)
5. Save.

**The Magic Result:** 
When you run this Concurrent Program from your custom HMS menu, Oracle EBS will fetch the raw SQL data as XML, silently pass it into your Microsoft Word RTF Template, and generate a perfectly styled PDF or Word Document for your mentor to view!
